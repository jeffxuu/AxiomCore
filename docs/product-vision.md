# 产品愿景

**Axiom Core — 个人决策智能核心**

> 焦土重建版（2026-05）。整套系统所有标识符（systemd、env、CSS、类型、API、数据库表）统一使用 axiom-core / `AXIOM_*`，**不保留 LifeOS 兼容名**。LifeOS 是已被废弃的前身仓库。

## 这是什么

Axiom Core 不是日历、不是打卡 App、不是个人 wiki。它是一个**约束驱动的决策引擎**：
- 输入：每日记录、健康数据、现金流、求职进度、学习投入、项目验证数据。
- 处理：用一组明文公理（公理 = Axiom）去推演当前阶段是否偏离主目标、是否触发风险预警。
- 输出：当前状态摘要、风险预警、明天的最重要一件事、本周需要否决的诱惑。

它的价值不在"更多记录"，而在**用机制对抗当下的情绪偏差**。

## 要解决的真实问题

当前所有者处于以下处境（这套系统的设计原点）：

1. **现金流脆弱**：债务还款 + 日常开支挤压个人现金流，剩余跑道以月计。
2. **求职窗口**：需要短期内拿到具备稳定现金流的岗位，避免动用业务种子资金救急。
3. **业务种子有限**：广州独立业务的可投入资金约 10 万元上下，必须严格隔离，禁止反向救济日常生活。
4. **健康下行风险**：作息、体重、心理状态有恶化迹象，长期失能会让所有其他规划失效。
5. **学习效率窗口**：英语、供应链、AI 工具是中期复合杠杆，但短期 ROI 低，容易在压力下被砍掉。

这五件事彼此挤压。Axiom Core 的工作是：在 AI 助手、日报、记事本、规划工具的喧哗中，**反复把注意力拉回到这五件事的优先级和约束上**。

## 核心闭环

```
飞书/网页录入  →  SQLite + Markdown 双写  →  自动汇总引擎
       ↑                                            ↓
  AI/CDO 决策建议  ←  风险预警 + ROI/Burn Rate 计算
```

每个闭环节点都可独立运行、独立调试：

- **录入层**：飞书 Bot、网页打卡、手动 Markdown 三条入口任选其一。
- **存储层**：SQLite 是运行态（不进 Git），Markdown 是长期可读快照（进 Git）。
- **汇总层**：`workflows/summarize_fragments.py` 在每日记录、财务看板、业务目标上扫一遍，输出预警与周报。
- **决策层**：人或 AI 读取 `profile/current-state.md`、`profile/constraints.md`、`profile/master-system-prompt.md`，按公理推演下一步动作。

## 当前阶段（截至重构时间点）

- 状态：**生存模式（survival mode）**。优先级链是：保命 > 现金流 > 求职 > 业务隔离 > 学习 > 长期规划。
- 数据：本地 + 云端双副本已跑通；飞书录入已上线；网页打卡已替代纯 Markdown 手录入。
- 缺口：决策引擎过去只是"提示"，不会真正"否决"。Axiom Core 阶段要让约束变成可执行的红线判定。

## 不做什么（防止 scope creep）

- **不做社交分享**。这是单人系统；多人协作、家庭账本、伴侣对账不在路线图内。
- **不做长期外部依赖**。能本地跑就本地跑，外部 SaaS 只用于飞书机器人这种用户端入口。
- **不做"AI 全能助手"**。Axiom Core 的 AI 是首席决策官，不是日程秘书、不是聊天伙伴。

## 与其他系统的边界

| 工具 | 角色 |
|------|------|
| Axiom Core | 决策、状态、风险预警 |
| 飞书 | 录入入口（机器人 + 文档） |
| GitHub | 版本回溯、跨设备备份 |
| 操作系统日历 | 真正的"日程提醒"——Axiom Core 不试图取代它 |
| 银行/支付 App | 真实金额来源——Axiom Core 只接收摘要数字 |

## 度量成功的方式

Axiom Core 自己**不被指标驱动**。它的成功标准在外部世界：

1. 现金流跑道 `cash.runway_months >= 6`
2. 业务种子资金 `business_cash.runway_months > 6` 且没有被挪用
3. 求职：每周投递 / 面试达到目标值
4. 健康：体重、睡眠、心率/血压在年度看板的安全区间
5. 长期：英语和供应链知识可以变现（出现在简历或合同里）

如果系统每天产出大量日志但这五项不变好，说明 Axiom Core 失败了，需要重新审视约束与人设而不是加更多功能。
