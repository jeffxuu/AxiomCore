# 06 — Cognition · 思维与灵感

> 认知升级、信息差捕获与高价值灵感。**未验证的想法的栖息地**——验证后才能升级到 `05_Projects/`。

## CDO 视角

Cognition 是 Axiom Core 里**最容易被误用的目录**。它**不是**：
- 不是个人成长鸡汤的剪贴板
- 不是"未来某天我要做的事"的许愿池
- 不是"今天读了什么好书"的读书笔记本

它**是**：
- 信息差捕获：你比市场早看到的、可能撬动决策的观察
- 灵感落地评估：把灵感拆成"假设 → 验证步骤 → 成本"
- 复盘洞察：从其他领域（健康/财务/求职）抽出的元规律

CDO 在这里做的事：**拒绝让 Cognition 成为逃避真实动作的避风港**。

## 录入规范

每条认知/灵感条目按以下模板：

```markdown
## YYYY-MM-DD · <一句话洞察>

**信号来源**：（哪本书 / 哪次对话 / 哪个数据观察）
**假设**：（如果这条洞察成立，会推导出什么）
**验证成本**：（多久能证伪 / 多少钱 / 多少精力）
**下一步动作**：（pending → 进入 05_Projects 沙盘 / 写入 09_Principles / 仅记录）
```

不写"下一步动作"的 Cognition 是**死记录**，3 个月后没人会回来读。

## 红线

| 触发 | CDO 动作 |
|------|---------|
| 同一灵感连续 3 个月写入但从未验证 | 强制评估：要么进沙盘要么删除 |
| 在 survival 模式下写"我打算辞职做 X" | 硬否决，要求先看 `02_Cashflow/` runway |
| 没有"信号来源"字段的条目 | 引擎跳过——只记录主观情绪不是认知 |

## 与决策引擎的关系

Cognition 不**直接**驱动 verdict，但它是灵感的容器：
- 进入 `05_Projects/` 沙盘的前置
- 沉淀到 `09_Principles/` 的素材库
- 复盘到 `08_Decisions/` 的对照镜

## 不录什么

- ❌ 摘抄+无评论 → 摘抄请收到 Readwise / Notion
- ❌ "心情不好" → 去 `01_Health/` 的 mood 字段
- ❌ 项目执行细节 → 去 `05_Projects/`
- ❌ 与人际有关的洞察 → 去 `07_Relationships/`
