# 04 — Skills · 技能与认知

> 技术栈与市场需求的匹配度。复利杠杆，不是兴趣爱好。

## CDO 视角

Skills 在 Axiom Core 里**不是"我喜欢学什么"**，而是**"哪些能力 3 年后还能用、且当前可以变现或降低风险"**。决策引擎 §3 ROI 评估的 `skill_compounding` 维度从这里读取数据。

**当前可见的高复利方向**（按优先级）：
1. 英语（外企岗位天花板、远程协作前置）—— 低成本/零成本约束
2. 全栈开发（React/TS/Python）—— 已具备，需要持续输出作品
3. AI Agent 编排（Claude Code / Cursor / 自动化）—— 当前窗口期，复利最强
4. 供应链 / 数据看板 —— 与现场管理经验复合，差异化优势
5. 项目管理 / 运营管理 —— 长期职业路径

不在以上清单的"想学"——先回答"3 年后还有用吗？" 答否则砍掉。

## 追踪的核心指标

| 指标 | 频率 | 红线 |
|------|------|------|
| `english_minutes`（英语学习分钟） | 日 | survival 期间必须零成本（墨墨/Anki/已有教材） |
| 当前主线技能（≤ 1） | 季 | 严禁同时学 ≥ 2 项 |
| 技能产出物（项目/文章/作品） | 月 | 长期复盘是否变成简历素材 |
| 市场需求验证 | 季 | JD 抓取、薪资中位数变化 |

## 录入规范

1. **学习时长 vs 学习产出**：录两个，前者证明投入，后者证明转化。
2. **每个技能附带验证场景**：英语→面试/简历/邮件；全栈→项目→简历素材；AI Agent→实际跑了什么自动化。
3. **不记"明天我想学的"**——只记今天实际做了什么。

## 与决策引擎的关系

[../docs/decision-engine.md](../docs/decision-engine.md) §3.1 ROI 评估的三个杠杆：
- `expected_value.skill_compounding` 0-3：未来三年此技能是否还产生现金流？
- `cost.time_per_week_hours`：与求职动作互斥
- `cost.cognitive_load`：与每日精力（`01_Health/`）的乘数关系
